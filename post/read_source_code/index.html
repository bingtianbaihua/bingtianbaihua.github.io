关于阅读开源代码后的思考

2016-07-02

Contents

实习以后，虽说做的东西不是太有技术含量，但多少写的代码还是很多很多的，比之前在网易的量大概要多三倍左右。

代码写的多了，偶尔有时间就会回头去看看。打开自己的代码一看，天呐，这特么是自己写的代码吗？扩展性呢？健壮性呢？高内聚低耦合呢？ 通通不见了，我甚至无法理解当时的我为什么会下此毒手写下这段糟糕的代码。于是开始重构。。。。。。

在公司里，也发现不少烂代码，不写注释不说，命名也是极其不规范。一个int 型的变量，硬是能取xx，ii这种名字，竟然还是一个号称工作了7年的“互联网老兵”，真是为之羞耻。对于此人的代码，我也看了不少，感觉确实是愧对自己的工作年限。比如：

public  void  calDailyJob(){
         if(getxxx()){
              //todo
        }else if(getxxx()){
               //todo
        }else if(getxxx()){
              //todo
        }
}

这种烂代码，光看就知道，此人不知道方法调用是需要耗费时间的，且没有太多的模块化思维，写的代码真是一坨屎。 这个时候明显可以将方法返回值传给一个变量。

算了，不说了。当我看了Apache common包下的几个类的源码，不能说颠覆了自己写代码的思维，但至少看了大牛是怎么最简化代码的。比如我举个例子，

     public  boolean   isBlank(String s){
              int len=0;
              if(s==null||(len=s.length)==0){
              }

              for(int i=0;i<len;i++){
                    //todo
              }
      }

这段代码，在判空的时候，就把字符串的长度保存在一个变量里，最小化了方法的调用，这才是好代码。类似于这种的例子，在开源代码里很多，我就不一一列举了，因为自己看的开源代码也只是少部分，要学的还有很多，要走的路还很长！！
